# Phase 3: Backend API Architecture (Model Serving)

## Goal
Build the translation layer that takes the highly complex data outputs from the `tft_best.ckpt` and the `master_feature_table` and converts them into cleanly structured JSON responses that a frontend web framework can safely digest and plot.

## Target Implementation Steps

### 1. Model Loading (FastAPI Scaffold)
- **Framework:** `FastAPI` (Python)
- **Logic:** Establish a server endpoint mapped to `src/serving/app.py`.
- **Pre-Cache:** The server will systematically load the PyTorch Checkpoint and GraphSAGE parameters directly into your computer's RAM upon booting up. This bypasses the heavy loading times, ensuring 5-second inference speeds during the actual Live Viva Demo. 

### 2. Probabilistic Translation Layer
- **Input Stream:** Takes requests for specific NDC (National Drug Code) or generic drugs directly from the front-end dashboard. 
- **Tensor Parsing:** Extracts the `P10/P50/P90` confidence values from the PyTorch inference tensors.
- **Payload Construction:** Wraps the arrays into standard 30-day forecast trajectories formatted as `{"expected_demand": [...], "worst_case_demand": [...]}`.

### 3. Rule-Based Shortage Detection 
- **The Shortage Trigger:** The Backend logic will computationally compare the TFT's P90 (Worst-Case) 30-day demand threshold against the supplier's static node topology constraints. 
- **The Flag:** If Demand > Supply, the Backend triggers an explicit `shortage_risk_flag = True` output.

### 4. Graph Topology Broadcaster
- Maps the structured edge layout generated by the `PyTorch Geometric` pipeline down to the frontend visualization library, including attaching the color-coded `risk_scores` directly to the JSON payloads representing individual suppliers.

## Final Output Structure
Generated a locally hosted web API running natively on `uvicorn` (localhost:8000), acting as the central intelligence hub for the upcoming Visual UI layer.
